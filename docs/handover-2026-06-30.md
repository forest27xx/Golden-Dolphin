## 五、S2 任务详细拆解

### Task #8: 扑克牌型系统（🔴 高优先级，关键路径起点）
来源策划案：docs/system-design/feishu-wiki/卡牌系统.md §2

需要实现：
- Suit 枚举（♠♥♦♣）和 Rank 枚举（2-A）
- PokerCard 数据结构（suit, rank, chipValue）
- 52 张标准扑克牌生成器
- PokerHandEvaluator：判定 9 种牌型
  - 高牌 / 对子 / 两对 / 三条 / 顺子 / 同花 / 葫芦 / 四条 / 同花顺
- 牌型基础积分表：高牌5 → 同花顺100
- 牌型基础倍率表：1x → 8x
- 牌面积分：2-10=面值, J/Q/K=10, A=11
- 初始手牌 8 张，基础出牌 3 次，弃牌 3 次
- 每次出牌 1-5 张，出牌后补牌，弃牌后补牌
- 替换现有 8 张效果牌系统（BuiltInConfigs + CardEffectResolver）

### Task #9: 坍塌值计算系统（🔴 高优先级）
来源策划案：docs/system-design/feishu-wiki/卡牌系统.md §3

公式：S_k = [(B_chips + ΣC_chips + F_chips) × (B_mult + A_mult) × ΠX_mult] + F_score

需要实现：
- 坍塌值计算器（替代 CollapseSystem.cs 的简单 Add(delta)）
- 参数：B_chips(牌型基础积分), ΣC_chips(牌面积分总和), F_chips(功能牌筹码加值)
- B_mult(牌型基础倍率), A_mult(功能牌倍率加法), X_mult(独立乘区), F_score(功能牌固定加值)
- S_total = ΣS_k（基础出牌累计）
- E_total = 节点抽出坍塌累计（等级1=100% Target, 等级2=10%, 等级3=5%）
- Collapse_total = S_total + E_total

### Task #10: 功能牌系统（🔴 高优先级，工时最大）
来源策划案：docs/system-design/feishu-wiki/卡牌系统.md §4-7 + 牌组系统.md §3-5

需要实现：
- FunctionCardLevel 枚举（等级1/2/3）
- FunctionCard 数据结构：名称、等级、解锁条件、主动效果类型、效果参数
- 主动效果执行：F_score 加值 / A_mult 加法 / 抽牌弃牌 / 重抽 / 链路标记
- 功能牌不占基础 8 张手牌，进入独立可操作区
- 等级1当前关抽出即通关（100% Target），来不及主动使用
- 跨关降级继承：同名同效果，降级进入后续关卡功能牌池
- L1 池 11 张（等级1×2, 等级2×4, 等级3×5），L2 池 22 张
- 替换 CardEffectResolver.cs 的硬编码逻辑
- 可拆分为三个子任务并行：数据定义 + 效果执行 + 继承机制

### Task #11: 建筑节点与 Slot 生成器
来源策划案：docs/system-design/feishu-wiki/牌组系统.md §6

需要实现：
- 建筑方块 → 功能牌节点映射
- L1=10 Slot 生成器（Rank1 0-1 / Rank2 3-4 / Rank3 5-6）
- L2=14 Slot 生成器（Rank1 0-1 / Rank2 5-6 / Rank3 7-8）
- 节点密度约束：max_instances_per_card=1, role_cap 1-2
- 继承占位策略：exclusive_replace
- 扩展 BuildingModel.cs 增加功能牌节点层

### Task #12: 存档系统升级
来源策划案：docs/system-design/feishu-wiki/存档系统.md

需要实现：
- 双备份机制：save_main + save_backup，写入先临时文件再替换
- 长期进度 / 设置项分区存储（新游戏不清设置）
- 首启 vs 续启主菜单状态切换
- 坏档处理：无法解析 / 版本不兼容 / 关键字段缺失 → 提示+恢复
- 新游戏二次确认弹窗
- P0 不做局内中断恢复
- 设置项：主音量 / BGM / SFX（与 Task #3 Audio 系统配合）

### Task #13: 回合状态机
来源策划案：docs/system-design/feishu-wiki/卡牌系统.md §8

需要实现：
- RoundPhase 枚举：抽牌 → 出牌/弃牌 → 功能牌使用 → 结算 → 补牌 → 判定
- 基础出牌次数 / 弃牌次数独立追踪（各 3 次）
- 功能牌主动使用不消耗基础出牌次数
- 结算顺序：出牌→判定牌型→读取功能牌修正→计算 S_k→更新 S_total→检查节点挑战→E_total→判定通关→补牌
- 替换 LevelManager.cs 中 outcome==Playing 的简单守卫
