# Game Research Notes - 2026-06-09

目标：为 2026-06-10 集中讨论准备游戏方向备选。筛选标准不是“游戏有多大”，而是：

- 核心玩法能不能在 24-48 小时内做出灰盒原型。
- 5 人团队能不能分工：程序、关卡/数值、美术/音效、文档/验收。
- 是否适合 Unity 3D 单机项目。
- 是否能从你已经跑通的 3D 贪吃蛇原型继续生长。
- 主题公布后是否容易换皮和解释。

## 一页结论

我建议明天优先讨论方向 A：

> 3D 贪吃蛇 + 收集成长 + 空间机关/路径规划。

理由：你昨天已经跑通了 3D 贪吃蛇原型，这是我们最大的确定性。它可以吸收 `Snake Pass` 的“蛇形身体与空间解谜”、`Katamari Damacy` 的“越滚越大/越吃越长”、`黄金矿工` 的“目标物价值选择”、`跳一跳` 的“短操作高反馈”。最终不一定叫贪吃蛇，可以根据主题包装成龙、列车、数据线、能量体、尾巴怪、贪吃机器人等。

备选方向：

| 优先级 | 方向 | 玩法关键词 | 适合程度 |
|---|---|---|---|
| A | 3D 贪吃蛇空间解谜 | 移动、成长、绕行、机关、收集 | 高 |
| B | 一键蓄力/时机挑战 | 蓄力、跳跃、落点、连击 | 中高 |
| C | 物理合成/吞噬成长 | 物理碰撞、合成、体积增长 | 中 |
| D | 自动战斗生存 | 移动走位、自动攻击、局内升级 | 中 |

## 1. 森林冰火人

![森林冰火人截图](https://imga999.5054399.com/upload_pic/2010/4/2/4399_1665489867.jpg)

- 来源：[4399 森林冰火人](https://www.4399.com/flash/20082.htm)
- 类型：平台跳跃 + 机关解谜。
- 介绍：玩家控制火人和冰人两个角色，通过不同属性的机关、门、按钮、平台与危险地形完成关卡。
- 玩法拆解：
  - 两个角色有不同属性限制。
  - 关卡中有按钮、机关门、移动平台、宝石收集。
  - 目标是让两个角色都安全到达出口。
- 可借鉴：
  - 将“双角色”改成“蛇头/蛇尾”“两种身体状态”“长身体压按钮”。
  - 用颜色或材质表达规则，降低教学成本。
  - 关卡可以灰盒先行，美术后补。
- 风险：
  - 如果做太多机关，关卡设计会吃时间。
  - 双角色控制不一定适合单机 3D，可以只借规则，不直接照搬。

## 2. 黄金矿工

![黄金矿工截图](https://imga999.5054399.com/upload_pic/2010/4/2/4399_1141331473.jpg)

- 来源：[4399 黄金矿工](https://www.4399.com/flash/1602.htm)
- 类型：时机点击 + 资源收集。
- 介绍：抓钩左右摆动，玩家在合适时机放下钩子，抓取不同价值和重量的物品，在限定时间内达到目标分数。
- 玩法拆解：
  - 核心输入只有一个：发射/抓取。
  - 目标物有价值、重量和位置差异。
  - 每局有时间限制和分数目标。
- 可借鉴：
  - 3D 贪吃蛇可以加入“目标价值选择”：吃近的小、远的大、危险的高分。
  - 可以做成短局：60 秒内吃够能量，越长越难转身。
  - 非常适合快速做出可验收原型。
- 风险：
  - 机制太简单，需要加入空间障碍、连击或目标变化。

## 3. 微信飞机大战

![微信飞机大战截图](https://imga999.5054399.com/upload_pic/2017/9/20/4399_09452197369.jpg)

- 来源：[4399 微信飞机大战](https://www.4399.com/flash/191910.htm)
- 类型：纵向射击 + 躲避 + 分数挑战。
- 介绍：玩家移动飞机躲避敌人和子弹，自动射击，收集道具并尽量获得高分。
- 玩法拆解：
  - 输入主要是移动。
  - 攻击自动发生，玩家注意力集中在走位。
  - 敌人按波次出现，道具提供短期强化。
- 可借鉴：
  - 如果做“蛇形生存”，可以把攻击交给身体或尾巴自动触发。
  - 波次生成和局内道具能增加内容感。
  - 适合做成 3D 俯视角竞技场。
- 风险：
  - 敌人、子弹、道具一多，调参和性能都可能变复杂。

## 4. 跳一跳

![跳一跳截图](https://i2.chinanews.com/simg/cmshd/2018/01/10/71e1e632cefa4191b78db98ddfb169d5.jpg)

- 来源：[维基百科：跳一跳](https://zh.wikipedia.org/wiki/%E8%B7%B3%E4%B8%80%E8%B7%B3)、[中新网截图报道](https://www.chinanews.com.cn/m/cul/2018/01-10/8420373.shtml)
- 类型：一键蓄力 + 精准落点。
- 介绍：玩家长按蓄力控制跳跃距离，让棋子连续跳到下一个平台。
- 玩法拆解：
  - 单输入，反馈清晰。
  - 玩家失败通常能立刻理解原因。
  - 平台距离和特殊平台形成变化。
- 可借鉴：
  - 可以作为备选：如果主题适合“跳跃/传递/连接”，做一键蓄力很稳。
  - 也可以嵌入 3D 蛇：蛇头蓄力冲刺，身体滞后形成风险。
- 风险：
  - 玩法太轻，必须靠主题包装、关卡变化和手感打磨。

## 5. 合成大西瓜

![合成大西瓜截图](https://upload.wikimedia.org/wikipedia/zh/8/82/%E5%90%88%E6%88%90%E5%A4%A7%E8%A5%BF%E7%93%9C%EF%BC%88%E5%B1%8F%E5%B9%95%E6%88%AA%E5%9B%BE%EF%BC%89.jpg)

- 来源：[维基百科：合成大西瓜](https://zh.wikipedia.org/wiki/%E5%90%88%E6%88%90%E5%A4%A7%E8%A5%BF%E7%93%9C)
- 类型：物理下落 + 同类合成。
- 介绍：玩家投放水果，相同水果碰撞后合成为更大的水果，空间被占满后失败。
- 玩法拆解：
  - 规则一句话能懂。
  - 物理碰撞带来不确定性和惊喜。
  - 合成链条提供长期目标。
- 可借鉴：
  - 可以做“蛇吃同类升级”：同颜色能量合并后变成更高价值目标。
  - 或做 3D 物理合成备选项目。
- 风险：
  - Unity 物理很容易出现穿模、抖动、不可控堆叠。
  - 如果主题不贴合，容易像换皮。

## 6. 羊了个羊

![羊了个羊截图](https://upload.wikimedia.org/wikipedia/zh/e/e9/Yanglegeyang.png)

- 来源：[维基百科：羊了个羊](https://zh.wikipedia.org/wiki/%E7%BE%8A%E4%BA%86%E4%B8%AA%E7%BE%8A)
- 类型：层叠消除 + 社交传播。
- 介绍：玩家从多层叠放的牌中选择可点击牌，凑齐三张相同牌消除，直到清空或槽位满。
- 玩法拆解：
  - 操作轻，失败明确。
  - “上层遮挡下层”的结构制造决策。
  - 难度和传播话题是核心。
- 可借鉴：
  - 可以借“遮挡层级”而不是借消除本身：3D 场景中目标被身体或障碍遮挡，需要规划顺序。
  - 也可作为 UI/关卡节奏参考。
- 风险：
  - 单纯复刻争议大；不建议作为主方向。

## 7. Snake Pass

![Snake Pass 截图](https://shared.akamai.steamstatic.com/store_item_assets/steam/apps/544330/ss_dac35e9f3ab3ffbcdf7b5f9b4f612650070198a6.1920x1080.jpg?t=1780400076)

- 来源：[Steam: Snake Pass](https://store.steampowered.com/app/544330/Snake_Pass/)
- 类型：3D 物理蛇 + 平台解谜。
- 介绍：玩家控制蛇在 3D 场景中滑行、缠绕、攀爬，完成收集和关卡目标。
- 玩法拆解：
  - 移动方式本身就是玩法，不是普通角色换皮。
  - 身体长度影响路径、转向和攀爬。
  - 关卡围绕“蛇能做什么”来设计。
- 可借鉴：
  - 这是当前 3D 贪吃蛇原型最值得看的 Steam 样本。
  - 我们不必做复杂攀爬，可以先做：吃东西变长、身体压按钮、绕柱子、不能撞自己。
  - 主题公布后，蛇可以包装成龙、线缆、能量流、队伍、列车。
- 风险：
  - 真正的蛇形物理和攀爬很难，不建议追求真实模拟。
  - 应采用“离散身体节点 + 平滑跟随”的可控实现。

## 8. Katamari Damacy REROLL

![Katamari Damacy REROLL 截图](https://shared.akamai.steamstatic.com/store_item_assets/steam/apps/848350/ss_6f8332bf0c21f7d5072e56655c3f77147c4832e7.1920x1080.jpg?t=1708327061)

- 来源：[Steam: Katamari Damacy REROLL](https://store.steampowered.com/app/848350/Katamari_Damacy_REROLL/)
- 类型：滚动收集 + 体积成长。
- 介绍：玩家推动球体在场景中滚动，吸附越来越大的物体，让球越滚越大。
- 玩法拆解：
  - 成长非常直观：从小物件到大物件。
  - 场景本身就是资源池。
  - 体积变大后能进入新的收集层级。
- 可借鉴：
  - 3D 蛇可以做“长度门槛”：长度达到 N 才能压住机关、围住目标、通过某类障碍。
  - 收集对象按大小/价值分层，增强成长感。
- 风险：
  - 场景资源量需求高；我们可以用低模几何体先做。

## 9. Vampire Survivors

![Vampire Survivors 截图](https://shared.akamai.steamstatic.com/store_item_assets/steam/apps/1794680/ss_6c55afe36be2a7784bf18cb9b3218321ae2d10e5.1920x1080.jpg?t=1778864981)

- 来源：[Steam: Vampire Survivors](https://store.steampowered.com/app/1794680/Vampire_Survivors/)
- 类型：自动攻击 + 生存 + Roguelite 升级。
- 介绍：玩家主要控制移动，攻击自动释放，在越来越密集的敌人中生存并升级。
- 玩法拆解：
  - 玩家只管走位，系统负责输出。
  - 局内升级带来强成长反馈。
  - 敌人密度和奖励循环推动继续玩。
- 可借鉴：
  - 可以做“蛇身自动攻击”：身体越长，攻击范围越大，但也越容易撞到自己。
  - 做 3D 俯视角竞技场时，内容扩展效率高。
- 风险：
  - 需要大量调参和反馈特效。
  - 如果敌人太多，Unity 性能和寻路会变成风险。

## 10. Dorfromantik

![Dorfromantik 截图](https://shared.akamai.steamstatic.com/store_item_assets/steam/apps/1455840/ss_a49e5fcc5301e8a05a1da7ce233a5377cf680e98.1920x1080.jpg?t=1780575985)

- 来源：[Steam: Dorfromantik](https://store.steampowered.com/app/1455840/Dorfromantik/)
- 类型：拼图建造 + 任务扩张。
- 介绍：玩家逐块放置地形拼图，让村庄、森林、河流、铁路等地块相邻组合，并完成任务获得更多地块。
- 玩法拆解：
  - 模块化内容，规则清楚。
  - 任务驱动扩张，节奏平和。
  - 每一局由随机抽取地块形成变化。
- 可借鉴：
  - 可以学习“任务卡”结构：收集 X 个、围住 Y 个、连接 A 到 B。
  - 如果做 3D 蛇关卡，任务系统可以让同一套移动机制产生不同目标。
- 风险：
  - 与 3D 贪吃蛇关系弱，只适合作为任务系统参考。

## 对我们项目的方向推演

### 方向 A：3D 贪吃蛇空间解谜

一句话：玩家控制一条会变长的 3D 蛇/龙/能量线，在场景中收集目标、压机关、绕障碍、避免撞到自己。

最小版本：

1. WASD 控制蛇头移动。
2. 身体节点跟随并随收集增长。
3. 吃到目标得分并变长。
4. 撞墙/撞自己失败。
5. 收集足够目标后出口开启。

可扩展：

- 长度压按钮。
- 绕柱子形成闭环。
- 不同颜色食物改变身体状态。
- 限时挑战或步数挑战。
- 关卡内有高价值但高风险目标。

为什么推荐：

- 复用你已经跑通的 3D 贪吃蛇基础。
- 核心机制一句话能懂。
- 灰盒阶段可玩，美术可以后置。
- 主题适配能力强。

### 方向 B：3D 吞噬成长

一句话：玩家控制一个会成长的主体，在场景中收集比自己小的物体，逐步解锁更大目标。

参考：`Katamari Damacy REROLL`、`合成大西瓜`。

优点：爽感强、容易展示。  
风险：物理和资源数量压力更大。

### 方向 C：蛇形生存竞技场

一句话：蛇越长越强，身体自动攻击或防御，但越长越难走位。

参考：`Vampire Survivors`、`微信飞机大战`。

优点：内容扩展快，战斗反馈强。  
风险：调参、敌人生成、特效和性能要小心。

### 方向 D：一键蓄力挑战

一句话：用最少输入完成连续跳跃/抓取/冲刺挑战。

参考：`跳一跳`、`黄金矿工`。

优点：开发稳、易上手。  
风险：如果没有主题包装和变化，容易显得薄。

## 明天讨论建议

每个人发言 3 分钟，按这个格式讲：

```text
我调研的游戏：
核心玩法一句话：
最值得借鉴的一点：
如果我们做，24 小时内能做出的灰盒版本：
最大风险：
我推荐/不推荐它作为主方向的原因：
```

讨论时不要先争美术风格，先确定三件事：

1. 核心动词：移动、吃、绕、跳、射击、合成、建造，选 1 个主动词。
2. 失败条件：撞自己、时间到、空间满、血量归零，选 1 个主失败。
3. 成长方式：变长、变大、升级、解锁地块，选 1 个主成长。

建议投票标准：

| 标准 | 权重 |
|---|---|
| 24 小时内能否做出可玩灰盒 | 30% |
| 是否能结合 3D 贪吃蛇原型 | 25% |
| 美术资源压力是否可控 | 15% |
| 是否容易贴合未知主题 | 15% |
| 是否有展示效果和传播点 | 15% |

## 我当前的推荐

主方向：`3D 贪吃蛇空间解谜`。  
一句话版本：

> 玩家控制一条不断变长的蛇/龙/能量线，在 3D 场景里吃掉目标、压住机关、绕开障碍，并在身体越来越长的限制下完成出口目标。

明天如果大家认可，可以立刻拆第一批 P0：

| P0 任务 | 验收标准 |
|---|---|
| 蛇头移动和镜头 | WASD 能稳定控制，镜头不晕 |
| 身体跟随和增长 | 吃一个目标增加一个身体节点 |
| 碰撞失败 | 撞墙或撞自己能触发失败 |
| 收集和出口 | 收集满 N 个目标后出口开启 |
| 第一张灰盒关卡 | 有障碍、目标、入口、出口 |
| UI 最小闭环 | 开始、重开、胜利、失败 |
