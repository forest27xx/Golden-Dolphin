# 方向提案：3D 贪吃蛇空间解谜

## 一句话概念

玩家控制一条会不断变长的 3D 蛇、龙或能量线，在立体关卡中收集目标、压住机关、绕开障碍，并在身体越来越长的限制下完成出口目标。

这个方向不是简单复刻传统贪吃蛇，而是把“身体变长”变成关卡机制：

- 越长，能力越强，可以压机关、围住目标、触发更高分奖励。
- 越长，移动越难，需要规划路线，避免撞墙或撞到自己。
- 每一关都可以围绕“长度带来的收益和风险”设计。

## 为什么优先考虑这个方向

我们昨天已经用 Unity 3D 跑通了 3D 贪吃蛇原型，这是目前最大的确定性。基于现有原型继续推进，比重新开一个完全不同的玩法更稳。

这个方向适合短周期比赛：

- 核心规则一句话能讲清楚。
- 24 小时内能做出可玩的灰盒版本。
- 不依赖大量美术资源，先用方块、圆球、胶囊体就能验证玩法。
- 适合 5 人分工：程序做移动和身体系统，关卡负责灰盒，美术音效后期替换，jun 负责玩法文档、验收和开发日志。
- 主题适配能力强：蛇可以根据主题包装成龙、列车、数据线、能量体、尾巴怪、贪吃机器人、队伍等。

### Snake Pass

来源：[Steam: Snake Pass](https://store.steampowered.com/app/544330/Snake_Pass/)

参考点：

- 角色移动方式本身就是玩法，不是普通角色换皮。
- 身体长度、转向、缠绕和空间路径会自然形成挑战。
- 关卡围绕“蛇能做什么”来设计。

对我们的启发：

- 不追求真实蛇形物理，避免开发过重。
- 用“离散身体节点 + 平滑跟随”实现稳定可控的蛇身。
- 先做吃东西变长、绕障碍、压按钮、撞自己失败这些核心体验。

- 收集目标可以分成普通目标、高价值目标、危险目标，形成路线选择。

## 核心玩法框架

### 玩家做什么

1. 控制蛇头在 3D 场景中移动。
2. 吃掉目标物，获得分数并增加身体长度。
3. 利用身体长度压机关、围住目标或解锁出口。
4. 避免撞墙、掉落或撞到自己的身体。
5. 在限定时间、限定步数或限定关卡目标内通关。

### 主要乐趣

- 成长感：身体越来越长，能力越来越强。
- 路线规划：同一个目标，走法不同，风险不同。
- 空间压迫：越强也越笨重，变长既是奖励也是负担。
- 关卡解法：用身体长度解决机关，而不是单纯跑酷。

### 失败条件

优先只保留一个主失败条件：

> 撞到墙、障碍或自己的身体。

后续可扩展：

- 掉出地图。
- 时间耗尽。
- 关键目标被错过。

### 成长方式

主成长方式：

> 吃到目标后增加身体节点。

可扩展：

- 特殊目标让身体变短、变硬、变快、变慢。
- 不同颜色目标改变身体属性。
- 连续吃目标获得 combo 分数。

## 最小可交付版本

第一版只追求“能玩、能输、能赢”：

| 模块 | 最小功能 | 验收标准 |
|---|---|---|
| 移动 | WASD 控制蛇头 | 操作稳定，镜头不晕 |
| 身体 | 身体节点跟随 | 吃 1 个目标增加 1 节身体 |
| 收集 | 普通目标物 | 碰到目标后消失、加分、变长 |
| 失败 | 撞墙/撞自己 | 能进入失败界面并重开 |
| 胜利 | 出口或目标数 | 收集满 N 个目标后通关 |
| 关卡 | 第一张灰盒地图 | 有墙、障碍、目标、入口、出口 |
| UI | 开始/胜利/失败/重开 | 主流程闭环可演示 |

## 后续扩展方向

如果第一版跑通，可以逐步加：

- 压力机关：长度足够才能压住按钮。
- 环绕目标：用身体围住能量核心后得分。
- 高风险奖励：危险区域里有高价值目标。
- 限时挑战：短时间内收集指定数量目标。
- 主题包装：根据比赛主题改成龙、线缆、列车、能量流等。
- 关卡组：教学关、机关关、限时关、综合关。

## 风险与控制

| 风险 | 控制方式 |
|---|---|
| 蛇身物理太难 | 不做真实物理蛇，用节点跟随实现 |
| 关卡设计太耗时 | 先做 1 张灰盒关卡，再复制改造 |
| 镜头容易晕 | 固定俯视或斜俯视角，不做自由 3D 镜头 |
| 美术来不及 | 先用低模几何体，后期统一材质和特效 |
| 玩法太像传统贪吃蛇 | 加入 3D 空间、机关、长度门槛和目标选择 |

## 推荐的第一周目标

在主题公布前，先把技术原型做扎实：

1. 3D 贪吃蛇移动稳定。
2. 身体增长稳定。
3. 撞自己和撞墙判定稳定。
4. 能在一张灰盒关卡里完成收集并通关。
5. 能打出一个可运行测试包。

主题公布后，再决定最终包装和关卡目标。

## 一句话推荐

我建议把这个作为主方向继续推进：

> 用 Unity 3D 做一个“会变长的蛇/龙/能量线”的空间解谜游戏。玩家通过收集让身体变长，用长度解决机关，同时也要承担身体变长带来的路线规划风险。
