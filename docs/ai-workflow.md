# AI 辅助开发规则

## Codex 适合做

- 生成 Unity C# 脚本初稿。
- 根据 Console 报错解释原因。
- 修改小范围脚本。
- 写 Editor 小工具。
- 拆解一个功能的代码步骤。

## 给 Codex 的请求模板

```text
Unity 版本：
目标功能：
已有脚本：
报错信息：
允许修改的文件：
验收标准：
```

## 不要这样问

```text
帮我做一个战斗系统。
```

## 要这样问

```text
Unity 版本：2022.3.74f1
目标：做一个最小可用的近战攻击脚本
输入：按 J 攻击
效果：检测玩家前方 1.5 米内 Enemy 标签对象，调用 TakeDamage(1)
限制：只改 PlayerAttack.cs，不改其它文件
验收：Console 无报错，测试敌人血量会减少
```

## 硬规则

- AI 写的代码不能直接合主分支。
- 必须在 Unity 里跑过。
- 每次只让 AI 改一个小目标。
- AI 产物要有人验收。
