## 1. 文档状态与适用范围
### 1.1 系统定位
存档系统负责自动保存、读取、重置和校验《塔楼牌 Turrot》的本地游戏数据，确保玩家在关闭游戏、通关关卡、失败重开、修改设置、返回主菜单等情况下，能够获得符合预期的进度连续性。
本系统不是玩法系统本身，而是所有长期进度与部分局内状态的底层支撑系统。
### 1.2 本系统负责
- 判断玩家是否首次启动游戏。
- 判断本地是否存在有效存档。
- 读取已有存档并恢复游戏进度。
- 创建新存档。
- 重置游戏进度。
- 保存已通关关卡。
- 保存已解锁功能牌。
- 保存已获得记忆碎片。
- 保存设置项。
- 根据 GDD 旧案 P2 内容，预留保存单局内牌库抽牌顺序、打出的牌、使用过的功能牌 buff 的扩展方向。
- 处理存档损坏、版本不兼容、读取失败等异常情况。
### 1.4 当前适用阶段
当前文档按 MVP / P1 本地单机版本整理。
已确认：MVP 使用本地存档即可。
建议：先实现单存档位，避免多存档位增加主菜单、覆盖确认、读档选择、测试矩阵和异常处理成本。
旧案来源：GDD 中“继续游戏”标为 P4。
当前系统案建议：MVP / P0 做“有无存档判断与主菜单入口状态切换”；“继续游戏恢复落点”作为待确认优先级，建议总控确认是否前置到 MVP。
待确认：继续游戏是否前置、是否支持局内中断继续、是否需要多存档位或云存档。
## 2. 系统目标
### 2.1 设计目的
存档系统需要保证玩家的关键进度不会丢失，同时避免失败、重开、新游戏等行为误伤长期进度。
核心目标：
- 首次启动时主菜单为“开始游戏 / 收藏室 / 设置 / 退出”
- 已有进度时主菜单切换为“新游戏 / 继续游戏 / 收藏室 / 设置 / 退出”的状态。
- 玩家点击新游戏按钮前会弹出窗口明确提示会清空已有游戏进度。
- 失败不清空总进度，只重置当前关卡临时状态。
- 设置项能独立保存，不应被普通失败重置误删。
### 2.2 玩家体验目标
体验
说明
连续性
关闭游戏后再次打开，能继续已有进度。
安全感
新游戏、坏档、版本不兼容等高风险情况有明确提示。
低打扰
普通保存尽量自动完成，不要求玩家频繁手动保存。

清晰边界
玩家能理解“设置”“收藏/图鉴”“当前关临时状态”哪些会保留、哪些会重置。
## 3. GDD 旧案来源摘要
### 3.1 初始逻辑
GDD 2.1.5“初始逻辑与存档判断”是旧案来源，提供了可搬运的主菜单初始逻辑。旧案中，玩家首次打开游戏时，主菜单显示：
- 新游戏：跳转到新手引导部分 / 关卡 1 的 Scene，并开始记录玩家游戏进度。
- 设置：调整音量和查看制作人员名单。
- 收藏室：按钮禁用，呈现灰色。
- 退出：关闭应用并退出到桌面。
### 3.2 后续逻辑
GDD 2.1.5 旧案中，玩家后续任何时候打开主菜单时，主菜单显示：
- 新游戏：清除玩家存档并跳转到新手引导部分。
- 新游戏需要询问玩家是否开始新游戏，因为该行为会清空已有的游戏进度。
- 继续游戏：根据游戏存档系统加载游戏进度。
- 设置：同首次启动。
- 收藏室：玩家点击后跳转至卡牌收藏室，可查看卡牌细节与收藏的记忆。
- 退出：同首次启动。
### 3.3 GDD“存档机制”口径
GDD“存档机制”旧案中写到：
- MVP 使用本地存档即可。
- 保存时机包括：完成关卡、进入下一关、修改设置或每次出牌后（P2）。
- 失败不清空总进度，只重置当前关卡临时状态。

### 5.2 推荐判断逻辑
```
读取存档文件
-> 文件不存在：返回无存档
-> 文件存在但无法解析：返回坏档
-> 版本字段不存在：返回坏档或旧档待迁移
-> 版本高于当前客户端：返回版本不兼容
-> 版本低于当前客户端：执行迁移
-> 校验关键字段
-> 校验通过：返回有效存档
-> 校验失败：返回坏档
```
### 5.3 继续游戏落点
GDD 旧案提到：若存在本地存档，继续游戏恢复到最近已完成关卡后的选择状态。
GDD 旧案同时提到：P2 可保存单局内状态。
当前系统案建议按阶段拆分：
- MVP / P0：只做有无存档判断，并据此切换主菜单入口状态；不要求真正恢复进度。
- 待总控确认：继续游戏恢复落点是否前置到 MVP；如果前置，建议先恢复到最近已完成关卡后的选择状态。
- 待总控确认：是否支持局内中断恢复；如果支持，再保存并恢复当前关卡局内状态。
说明：GDD 的 P4 / P2 仅作为旧案优先级参考，不作为当前系统案的最终排期权威。

### 8.3 保存失败处理
建议：
- 设置保存失败：提示“设置保存失败，本次修改可能不会保留”。
- 进度保存失败：提示“进度保存失败，请重试或返回主菜单”。
- 关卡通关保存失败：不应直接进入下一关，避免奖励丢失。
- 自动保存失败：记录错误，必要时弹出轻提示。
## 9. 读档 / 新档 / 无档 / 坏档处理
### 9.1 无档处理
系统表现：
- 显示“开始游戏”。
- “继续游戏”不显示
- 收藏室置灰，不可交互
点击开始游戏：
1. 创建新存档。
1. 进入新手引导 / 关卡 1。
1. 开始记录游戏进度。
### 9.2 有档读档处理
系统表现：
- 显示“新游戏”。
- 显示“继续游戏”。
- 收藏室可点击。
点击继续游戏：
1. 读取存档。
1. 校验版本和字段。
1. 恢复最近进度点。
1. 若总控确认实现继续游戏，建议默认进入最近已完成关卡后的选择状态；若另行确认局内中断继续，再按局内状态恢复。
### 9.3 新游戏处理
点击“新游戏”后必须二次确认：
```
开始新游戏会清空当前游戏进度，是否继续？
确认 / 取消
```
确认后：
- 清除长期进度。
- 清除当前局内状态。
- 重新创建默认进度。
- 进入新手引导 / 关卡 1。
- 设置项是否保留：建议保留，待确认。
### 9.4 坏档处理
坏档包括：
- 文件不存在但系统误判有档。
- 文件无法解析。
- 关键字段缺失。
- 字段类型错误。
- 校验码不一致。
- 版本迁移失败。

- save_backup
保存新进度时先写临时文件，再替换主存档；替换成功后更新备份，降低写入中断造成坏档的概率。
## 10. 与其他系统的关系
### 10.1 与主菜单的关系
主菜单依赖存档系统提供：
- 是否首次启动。
- 是否存在有效存档。
- 是否可继续游戏。
- 收藏室是否可用。
- 新游戏是否需要清档确认。
主菜单操作会触发：
- 开始游戏：创建新档。
- 新游戏：清除旧进度并创建新档。
- 继续游戏：读取进度。
- 设置：读取和保存设置。
- 收藏室：读取已解锁条目。
### 10.2 与设置系统的关系
设置项属于存档系统的一部分，但建议与游戏进度分区。
原因：
- 玩家新游戏通常不希望主音量、音乐音量、音效音量、震屏、跳过已读剧情等偏好被重置。
- 设置项需要在无进度时也能保存。
- 设置项保存失败不应影响关卡进度。
### 10.3 与局内流程的关系
局内流程需要存档系统支持：
- 完成关卡后保存通关进度。
- 失败后清理当前关临时状态，但不清空总进度。
- P2 如支持局内中断继续，每次出牌后保存局内状态。
当前建议：
- P1 不强制支持局内中断继续。
- P2 再加入每次出牌后的完整局内快照。
### 10.4 与关卡进度的关系
关卡系统需要读取：
- 已通关关卡。
- 当前可进入关卡。
- 最后一关是否解锁。
- 当前关卡是否已领取奖励。
存档系统不决定关卡解锁规则，只保存关卡系统计算后的结果。
### 10.5 与卡牌 / 牌组 / 功能牌系统的关系
本节对齐卡牌系统 v0.2 revision 557、牌组系统、塔楼关卡系统与 CARD 系列拍板项。这里记录“存档边界”：玩法规则按当前拍板口径处理，保存粒度、字段结构和恢复时机仍待程序确认。
已确认 / 稳定边界：
- 基础牌库固定为 52 张标准扑克牌。
- 功能牌不混入基础牌库，而是由建筑功能牌节点抽出。
- 功能牌抽出后进入玩家可操作区 / 手牌，可主动打出。
- 功能牌主动打出不消耗基础出牌次数。
- 当前累计总坍塌值使用 Collapse_total 口径。
局内状态：仅在支持局内中断恢复时需要保存，不进入默认 P0：
- 当前基础抽牌堆顺序。
- 当前手牌。
- 已打出 / 已弃掉基础牌。
- 剩余出牌次数、剩余弃牌次数。
- 已抽出功能牌。
- 已消耗功能牌。
- 等待生效功能牌效果。
- S_total / E_total / Collapse_total。
- 建筑节点已抽出状态。
长期 / 局外状态：规则已拍板的功能牌池、等级一降级 / 继承按当前口径进入存档边界；具体保存粒度、字段结构和写入时机待程序确认：
- 已解锁功能牌。
- 已见过功能牌。
- 功能牌收藏 / 图鉴显示状态。
- 关卡功能牌池：对齐 Q-CARD-005，已拍板为“每关有功能牌池；后一关可基于前一关池加入新功能牌；第一关等级一降级后进入后续关卡池”。
- 等级一功能牌通关后降级：对齐 Q-CARD-004，规则已拍板；降级目标、字段结构和保存粒度待程序 / 配置确认。
- 等级一功能牌继承到后续关卡：对齐 Q-CARD-004，规则已拍板；由 RewardSystem 在通关后处理功能牌池继承 / 更新。
- 记忆碎片、收藏 / 已读内容继续沿用本存档系统既有长期进度口径。
需要与 Q-CARD 拍板口径保持一致：
- Q-CARD-003：E_total 已进入首版数值模拟和程序实现，参与 Collapse_total = S_total + E_total。
- Q-CARD-004：等级一 100% 坍塌进入当前规则；发生在节点抽出 / E_total 阶段，通关后的降级、继承、奖励和功能牌池更新由 RewardSystem 处理。
- Q-CARD-005：每关有功能牌池，后一关可基于前一关池加入新功能牌；第一关等级一降级后进入后续关卡池，不等于“第一关抽出的所有功能牌必定进入第二关”。
- 既有存档问题：是否支持局内中断恢复；若不支持，上述局内运行态不需要落入首版存档。
### 10.6 与收藏室 / 图鉴 / 收集系统的关系
收藏室至少需要读取：
- 已获得记忆碎片。
- 已解锁功能牌。
- 已见过卡牌详情。
建议：
- 收藏室数据属于长期存档。
- 失败不清空收藏室数据。
- 新游戏是否清空收藏室数据待确认；若“新游戏”定义为清空游戏进度，则默认会清空，除非后续定义“收藏永久保留”。
### 10.7 与 KidsHub / 夜航报告的关系
当前已读取资料中未找到 KidsHub / 夜航报告的明确口径。
建议：
- 若 KidsHub / 夜航报告属于局外长期内容，应进入长期存档。
- 若它们只是外部平台或运营入口，应单独建模块，不混入核心进度存档。
待确认：
- KidsHub 是否为当前项目正式系统。
- 夜航报告是否与记忆碎片 / 收藏室有关。
- 是否需要支持跨局保留。

## 14. 验收标准
### 14.1 首次启动
- 无存档时显示“开始游戏”。
- 无存档时“继续游戏”不可用。
- 无存档时收藏室禁用。
- 点击开始游戏后创建新存档并进入新手引导 / 关卡 1。
### 14.2 后续启动
- 有有效存档时显示“新游戏”和“继续游戏”。
- “继续游戏”恢复最近已完成关卡后的选择状态的优先级待总控确认；若未前置，P0 只验收入口状态正确。
- 收藏室可进入，并能看到已解锁内容。
### 14.3 新游戏
- 点击新游戏必须出现清档确认。
- 取消后原存档不变。
- 确认后清除游戏进度并进入新手引导 / 关卡 1。
- 设置项是否保留按最终确认口径执行。
### 14.4 进度保存
- 通关后重启游戏，已通关关卡仍保存。
- 解锁功能牌后重启游戏，解锁状态仍保存。
- 获得记忆碎片后重启游戏，记忆碎片仍保存。
- 修改设置后重启游戏，设置项仍保存。
### 14.5 失败处理
- 当前关失败后，总进度不清空。
- 当前关失败后，当前关临时状态重置。
- 失败后可重新开始当前关或退出到主菜单。
### 14.6 异常处理
- 坏档不会导致游戏卡死。
- 版本不兼容有提示。
- 保存失败有提示或错误处理。
- 主存档损坏时，如存在备份，可尝试恢复。
