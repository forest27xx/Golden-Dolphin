# 3. 主菜单界面
## 3.1 界面定位
主菜单是玩家进入游戏后的第一个可交互界面，承担开始游戏、继续游戏、设置、收藏室、制作人员和退出功能。主菜单需要建立暗色、神秘、塔楼、月亮、云雾和记忆碎片相关的整体氛围。
## 3.2 基础布局
- 画面上方或视觉中心展示游戏标题 Logo。
- 背景为暗色塔楼、月亮、云雾、星光或碎片元素。
- 主菜单按钮集中排列在画面左侧或视觉稳定区域。
- 按钮需要有普通、悬停、点击、禁用状态。
- 继续游戏、收藏室等按钮需要根据解锁或存档状态显示可用 / 不可用。

## 3.4 新游戏与继续游戏
详细参考存档系统
### 3.4.1 新游戏
- 玩家点击新游戏后，如果不存在旧存档，直接进入新手引导或第一关。
- 如果存在旧存档，弹出确认窗口，提示该操作会覆盖已有进度。
- 确认后进入新手引导或第一关；取消则关闭确认窗口，返回主菜单。
### 3.4.2 继续游戏
- 继续游戏按钮只负责作为入口，不在 UI 文档内展开存档读取细节。
- 读取成功后进入对应关卡或战斗界面。
- 读取失败时，显示轻量错误提示，并提供返回主菜单或重新尝试。

收藏室是主菜单的一个子模块。它主要用于查看已解锁的卡牌、记忆内容和故事回顾，不写数值、不写卡池。

## 3.7 制作人员窗口
制作人员窗口是主菜单下的子窗口，用于展示项目参与人员名单。它不进入新的游戏流程，也不改变存档或关卡状态。
### 3.7.1 入口与关闭
- 入口：玩家在主菜单点击“制作人员”按钮。
- 打开方式：在主菜单上方弹出制作人员窗口，背景主菜单半透明遮罩。
- 打开时窗口外无法点击操作
- 关闭方式：点击关闭按钮或者空白热区。
- 关闭后回到主菜单，主菜单按钮恢复可操作状态。
### 3.7.2 窗口布局
- 窗口标题显示“制作人员”。
- 名单区域居中或左对齐展示，按职责分组。
- 底部固定放置“返回”按钮。
- 名单内容超出一屏时，名单区域支持纵向滚动；返回按钮不随滚动消失。
- 窗口宽度不宜过大，避免遮住主菜单整体氛围；但文字需要保证可读。

### 3.7.4 交互状态
- 制作人员按钮：普通、悬停、点击三种状态。
- 窗口打开时：主菜单其余按钮不可点击，避免玩家误触。
- 返回按钮：普通、悬停、点击状态。
- 滚动条：仅在名单超出显示区域时出现。
### 3.7.5 表现要求
- 整体风格需要和主菜单一致，保持暗色、塔楼、月光、碎片等视觉语气。
- 文字优先保证清晰可读，不做过强装饰。
- 制作人员窗口有轻量打开 / 关闭动效，具体动效和音效写入转场与动效音效需求清单。
- 窗口不需要复杂交互，不需要分页；名单过长时使用滚动即可。
## 3.8 退出确认窗口
- 点击退出后弹出确认窗口。
- 确认：退出游戏或返回平台规定的退出态。
- 取消：关闭窗口并返回主菜单。
# 4. 新手引导界面
## 4.1 界面定位
新手引导只承担基础操作教学，让玩家理解如何选牌、出牌、弃牌，以及功能牌如何解锁和使用。存档读取到玩家第一次进入首关，会弹出新手引导窗口。窗口界面有一下信息以及按钮：
1. 图片：游戏内画面截图解释每一步骤的画面展示
1. 文本描述：详细描述怎么操作。
1. 切到上一页按钮：最后页面不包含，其他窗口包含。点击切换到下一页。
1. 切到下一页按钮：首个页面不包含，其他窗口包含。点击切换到上一页。
1. “知道了”ui按钮：只在特殊页显示。点击关闭新手引导窗口，进入游戏流程。

## 5.4 建筑与功能牌展示
- 建筑由功能牌或功能牌节点构成，是战斗界面的主要观察对象。
- 玩家可以悬停并点击建筑中的功能牌，查看功能牌详情。
- 详情浮层包含名称、解锁条件和简短说明。
- 浮层需要避开顶部状态栏、战斗结算表现区和手牌区。
## 5.5 手牌区
抽取的功能牌和基础牌都包含在手牌区。功能牌与基础牌的视觉区分来自卡牌本身的设计不同，不需要额外依赖复杂状态框来区分。通过按键切换显示所有手牌，仅显示功能牌和仅显示基础手牌
### 5.5.1 手牌显示
- 手牌区显示玩家当前拥有的可打出或弃置的牌。
- 基础牌使用基础牌面。
- 功能牌使用不同于基础牌的牌面设计，让玩家一眼能区分。
- 功能牌没有独立可操作区，也没有功能牌弃置区。
### 5.5.2 手牌交互状态
- 未选中：卡牌保持常态位置。
- 选中：卡牌上浮、并高亮描边。
- 取消选中：再次点击卡牌会取消选中处于选中状态中的卡牌，点击空白区域会取消选中所有处于选中状态中的卡牌。
- 被打出：所有处于选中状态中的卡牌离开手牌区，进入动态算分舞台。
- 被弃置：所有处于选中状态中的卡牌离开手牌区，播放弃牌动画。

### 5.5.2功能牌
玩家可通过悬停查看塔牌、手牌区功的能牌详细描述。

## 5.6 功能牌打出后的状态查看栏——buff栏
功能牌从手牌区打出后，如果产生持续效果、待触发效果或其他buff等状态，将出现在buff栏。
- buff栏需设在画面中的左侧或顶部空间。状态内容需要直接展示给玩家。玩家无需点击或其他操作就应能直接查看当前状态情况。（3-4条）
- buff栏不占用手牌区空间且不作为新的功能牌操作区。
- 区域展示当前正在生效的状态及其内容。生效结束后内容将自动消失，区域保留。
## 5.7 出牌后算分区
玩家出牌后的手牌进入算分区。算分表现模块位于画面中间。玩家点击出牌后，所选卡牌进入该区域，界面展示本次出牌结果、坍塌值变化、及功能牌触发反馈。
- 卡牌从手牌区进入中间算分表现区。
- 中间区域显示本次出牌后卡牌的动效。
- 坍塌值变化同步反馈到顶部状态栏。
- 如果触发功能牌状态，则同步更新顶部或左侧的功能牌状态小方框。

## 5.9 牌库
牌库区是战斗界面中的查看入口，用于让玩家确认当前牌库还剩哪些牌。该模块只负责前端展示与查看交互，不负责抽牌规则、牌库初始化或牌库顺序逻辑。
### 5.9.1 默认显示
- 牌库区是战斗界面中的可点击入口，默认只显示牌库入口本身。
- 入口可以表现为牌库卡背、牌堆图标或“牌库”短标识。
- 默认状态不展开显示任何剩余卡牌明细，也不直接展示具体牌面。
- 如界面需要显示剩余数量，只显示“剩余 X 张”一类数字信息，不展示卡牌列表。
- 入口需要有可点击感，鼠标悬停时可出现轻微高亮或提示。
### 5.9.2 点击后弹窗
- 玩家点击牌库入口后，弹出牌库窗口。
- 弹窗用于查看当前牌库中还剩哪些牌。
- 弹窗只承担查看功能，不允许玩家在窗口内抽牌、选牌、排序、删除、编辑或改变牌库顺序。
- 弹窗打开时，战斗界面保留在背景层，可根据视觉需要做轻微压暗。
- 玩家点击关闭按钮、点击窗口外遮罩区域或按返回键后，关闭弹窗并回到战斗界面。
